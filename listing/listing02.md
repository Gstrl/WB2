Что выведет программа? Объяснить вывод программы. Объяснить как работают defer’ы и их порядок вызовов.

```go
package main

import (
	"fmt"
)


func test() (x int) {
	defer func() {
		x++
	}()
	x = 1
	return
}


func anotherTest() int {
	var x int
	defer func() {
		x++
	}()
	x = 1
	return x
}


func main() {
	fmt.Println(test())
	fmt.Println(anotherTest())
}
```

Ответ:
```
2
1

Отложенная функция: defer указывает на выполнение отложенной функции 
после завершения основной функции, но перед фактическим возвращением 
значения.

В test отложенная функция изменяет именованное возвращаемое значение, 
и это изменение учитывается при возвращении из функции.
Когда функция объявляет возвращаемые значения с именами, как в случае
с функцией test, Go создает переменные с этими именами. Эти переменные 
существуют в области видимости всей функции, и они остаются доступными 
для изменения до тех пор, пока функция не завершится, включая выполнение 
отложенных функций.

В anotherTest отложенная функция изменяет переменную x, но это изменение 
происходит после того, как значение для возвращения уже запомнено.
Если возвращаемые значения не именованы, текущее значение возвращаемых 
переменных запоминается перед выполнением отложенных функций, и изменения, 
сделанные отложенными функциями, не влияют на возвращаемое значение.
```
